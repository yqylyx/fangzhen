# Phys-HPGT 方法设计与数据处理详解

本文档针对当前主项目的 `PhysHPGT` 模型做一份偏“方法说明书”的整理，重点解释四件事：

1. 这个模型要解决什么问题。
2. 原始 `.npy` 数据是怎样被读入、清洗、标准化并拼成 batch 的。
3. 模型内部是怎样从“电压时序 + 功角时序”一步步得到最终分类结果的。
4. 训练时除了二分类，还额外做了哪些辅助目标，为什么这样做。

文档基于当前仓库主项目代码实现整理，对应核心文件主要是：

- `src/data/dataset.py`
- `src/data/normalization.py`
- `src/models/phys_hpgt.py`
- `src/models/losses.py`
- `src/train.py`
- `src/eval.py`
- `configs/config.yaml`

## 1. 任务背景与当前实验协议

这个项目的主任务是做电力系统暂态稳定诊断，输出是一个二分类结果：

- `0 = stable`
- `1 = unstable`

除了最终分类，模型还会额外预测一个连续风险值 `risk in [0, 1]`。这个风险值不是最终标签本身，而是一个辅助学习目标，用来把“接近失稳”的样本也纳入学习过程，让模型学到更平滑、更符合物理规律的判别边界。

当前配置文件 `configs/config.yaml` 对应的是一个明确的跨系统 OOD 协议：

- 训练集来自 `36data`
- 验证集从 `36data` 内部随机切出 `10%`
- OOD 测试集使用 `37data`
- OOD 测试集使用 `74data`

也就是说，模型不是在 36、37、74 混合随机抽样训练，而是故意只在一个系统上训练，再到另外两个系统上验证泛化能力。这种设置更能检验模型是否真正学到了跨系统稳定性特征，而不是只记住某个固定系统的数据分布。

## 2. 原始数据格式与读取方式

### 2.1 单个样本的组织形式

每个样本是一个 `.npy` 文件，内部应当是一个字典。代码兼容以下字段：

- 时间轴：`t` 或 `times`
- 电压矩阵：`V` 或 `voltages`
- 功角矩阵：`delta` 或 `angles`
- 类别标签：`y` 或 `label`
- 其他信息：`meta`

典型形状可以理解为：

- `t: [T]`
- `V: [T, Nv]`
- `delta: [T, Ng]`
- `y: scalar`

其中：

- `T` 是时间长度
- `Nv` 是电压通道数
- `Ng` 是功角通道数

这里最重要的一点是：`T`、`Nv`、`Ng` 都不要求在不同样本之间一致。模型从一开始就是按“可变长度、可变通道数”的思路设计的。

### 2.2 数据字段兼容与形状修正

`dataset.py` 在读取数据时会做几步兼容处理：

1. 如果 `np.load` 出来的对象是标量 object 数组，会先 `.item()` 还原成字典。
2. 如果 `V` 或 `delta` 是一维数组，会自动升成二维。
3. 如果发现数组的时间维和 `t` 的长度不一致，但转置后能对齐，就自动转置。
4. 如果缺少 `t`，代码会用 `0, 1, 2, ..., T-1` 自动构造一个时间轴。

这几步的目标很明确：尽量兼容早期数据格式差异，但不悄悄吞掉真正的数据错误。如果时间维始终对不上，代码会直接报错。

## 3. 数据预处理流程

## 3.1 缺失值与哨兵值处理

配置里定义了一个哨兵值：

- `sentinel: -99999.0`

读取样本后，代码会把下面两类值都视为无效值：

- 非有限值，比如 `NaN`、`inf`
- 等于哨兵值 `-99999.0`

处理结果分成两部分：

- `clean` 数组：无效位置替换成 `NaN`
- `valid mask`：布尔掩码，记录哪些位置是真实有效观测

这样做的好处是：

- 数值张量本身可以继续用于归一化和网络计算
- 有效性信息不会丢失，后续所有 pooling、attention、图计算都可以显式跳过无效位置

### 3.2 功角预处理

功角 `delta` 的默认预处理模式是：

- `initial_then_mean`

它包含两步：

1. 每个功角通道减去该通道第一个有效时刻的值。
2. 每个时刻再减去该时刻所有有效功角通道的均值。

直观理解：

- 第一步去掉每个通道自己的初始偏置，让模型更关注相对变化量。
- 第二步去掉同一时刻整体角度平移，让模型更关注通道间相对分离，而不是绝对角度零点。

对暂态稳定任务来说，这样的处理非常合理，因为失稳往往体现在角度分散、相对摆动和后续尾部残留，而不是某个绝对角度原点。

### 3.3 为什么同时保留原始值和模型输入值

数据集返回时，会同时保留两套张量：

- `raw_V`、`raw_delta`
- `V`、`delta`

区别是：

- `raw_*` 是“完成缺失值处理和角度预处理后，但未做归一化”的版本
- `V`、`delta` 是“送进模型的归一化版本”

保留 `raw_*` 的原因是，后面的物理风险特征提取要依赖原始量纲。比如：

- 电压低于 `0.9`、`0.85`
- 功角扩散是否超过 `120°`、`180°`

这些阈值只有在原始物理量上才有意义，不能在归一化空间里直接判断。

### 3.4 归一化策略

归一化由 `TransientNormalizer` 完成，核心原则是：

- 只用训练域的训练子集拟合统计量
- 验证集与 OOD 测试集只能复用训练集统计量
- 电压和功角分开拟合，不混在一起

当前默认方法是：

- `method: robust`

对应的统计量是：

- 中心值使用 `median`
- 缩放值使用 `IQR = Q3 - Q1`

这是比均值方差更稳健的做法，原因是暂态电力数据通常存在异常波动、极端扰动、类别不平衡和截断片段，robust 统计更不容易被少量极端点拉偏。

另外，归一化器不会简单遍历所有值全量拟合，而是会做采样控制：

- 每种模态最多采样 `max_values_per_modality`
- 每个样本也有单独采样上限

这样可以避免大数据量下统计拟合过慢，同时仍保留整体分布特征。

### 3.5 Batch 拼接与 mask 设计

不同样本的 `T`、`Nv`、`Ng` 不相同，所以不能直接堆叠。`collate_fn` 会把一个 batch 补齐到：

- 时间维最大长度 `max_t`
- 电压通道最大数 `max_nv`
- 功角通道最大数 `max_ng`

补齐后的 batch 里包含多种 mask：

- `mask_V: [B, T, Nv]`
- `mask_delta: [B, T, Ng]`
- `ch_mask_V: [B, Nv]`
- `ch_mask_delta: [B, Ng]`
- `time_mask: [B, T]`

这些 mask 的作用分别是：

- `mask_V` 和 `mask_delta` 标记某个时间点、某个通道的数据是否真实有效
- `ch_mask_V` 和 `ch_mask_delta` 标记这个样本里该通道是否真实存在
- `time_mask` 标记这个时间步是否存在任何有效观测

这套设计非常关键。因为这个模型不是通过“把所有样本裁成完全一样的 shape”来解决问题，而是通过“padding + mask”保留真实结构，再让后续模块显式感知哪些位置是有效的。

## 4. 整体模型设计思路

`PhysHPGT` 这个名字可以理解为一个“物理感知的、层次化的、图增强的时序模型”。它的整体思路不是把所有通道简单拼在一起做一个普通时序分类器，而是分成几个更有结构感的阶段：

1. 每个通道先独立做时间 patch 编码。
2. 每个通道内部先学习时间动态。
3. 再把每个通道看成一个节点，建立跨通道关系图。
4. 电压分支和功角分支分别编码后再融合。
5. 用多个出口同时监督，让浅层和深层都学到可判别表示。

这样做的动机是：

- 暂态稳定问题有明显的时间结构
- 通道之间有显著耦合关系
- 电压和功角在物理语义上并不完全相同
- 不同系统的通道编号和通道数量可能不一致，所以不能写死拓扑或通道索引依赖

## 5. 输入到输出的张量流

可以把一次前向传播理解成下面这条主线：

1. 输入：
   `V: [B, T, Nv]`
   `delta: [B, T, Ng]`

2. 按通道切时间 patch 后，得到：
   `v_tokens: [B, Nv, P, D]`
   `d_tokens: [B, Ng, P, D]`

3. 每个通道沿 patch 序列做时间编码后，沿 patch 维做池化，得到：
   `v_nodes: [B, Nv, D]`
   `d_nodes: [B, Ng, D]`

4. 把所有通道节点拼成一个节点集合：
   `all_nodes: [B, Nv + Ng, D]`

5. 先做一次跨模态融合，得到一个全局决策状态：
   `state_0: [B, D]`

6. 再经过若干层图注意力，每一层后再做一次融合，得到：
   `state_1, state_2, ...`

7. 把这些状态整理成 `n_exit=3` 个出口，每个出口各自输出：
   `logits: [B, 2]`
   `risk: [B, 1]`

8. 最终分类评估默认取最后一个出口的结果。

## 6. 时间 patch 编码

### 6.1 为什么先切成 patch

原始时间长度 `T` 可能很长，如果每个通道直接逐时刻进 Transformer，计算量会很大。模型因此先把每个通道沿时间维切成 patch：

- `patch_len = 16`

也就是说，每 16 个时间步组成一个局部时间块，再把这个时间块映射成一个 token。

这样做有几个直接好处：

- 降低后续时序编码的序列长度
- 让模型先在局部时间窗上做摘要
- 对局部扰动、震荡片段和短时电压跌落更敏感

### 6.2 patch 有效性控制

不是所有 patch 都一定有效，因为有些时间段可能存在缺失或 padding。代码会先计算每个 patch 的有效比例：

- `valid_ratio = valid_count / patch_len`

只有当有效比例至少达到：

- `patch_valid_ratio = 0.25`

这个 patch 才被视为有效 token。

这意味着：

- 大量缺失的 patch 不会强行参与后续时序建模
- 有助于提升模型对截断、不齐整采样和 padding 区域的鲁棒性

### 6.3 patch 向量是如何构造的

每个 patch 内部，模型先对有效时间点求均值，再乘回 `patch_len`，本质上是把均值摘要映射回一个近似“局部总量级别”的表示。之后再通过：

- `Linear(patch_len -> d_model)`
- `LayerNorm`
- `Dropout`

投影成统一维度 `d_model=128` 的 token。

这里有一个很重要的设计选择：patch 编码是“按通道独立进行”的，而不是把所有通道揉成一条序列。这样一来，模型对通道编号没有强绑定依赖，更适合跨系统泛化。

## 7. 时序编码模块

### 7.1 模态分离

电压和功角各自有一套 patch token：

- 电压分支 `v_tokens`
- 功角分支 `d_tokens`

模型给两种模态分别加上一个可学习的 `modality embedding`，让后续模块知道“这个 token 来自电压还是功角”。

### 7.2 位置编码

每个 patch 序列都会加上标准的正弦位置编码。作用很直接：

- patch 序列本身有先后顺序
- 模型需要区分“前期扰动”和“后期恢复/失稳”

### 7.3 Transformer 或 TCN

代码支持两种时间编码器：

- `transformer`
- `tcn`

当前配置使用的是：

- `temporal_encoder: transformer`

也就是每个通道的 patch 序列会独立通过若干层多头自注意力和前馈网络。

这里的自注意力是“每个通道内部沿时间维”的，不是在通道之间做。这样建模的含义是：

- 先让每个电压通道学会自己的动态演化
- 先让每个功角通道学会自己的动态演化
- 通道之间的关系，后面再交给图注意力建模

这是一种很清楚的“先时间、后通道关系”的分层结构。

## 8. 从时间 token 到通道节点

一个通道经过时序编码后，仍然是一个 patch 序列。接下来模型对 patch 维做 mask-aware 平均池化，得到每个通道一个节点向量：

- 电压通道变成电压节点
- 功角通道变成功角节点

得到的结果是：

- `v_nodes: [B, Nv, D]`
- `d_nodes: [B, Ng, D]`

这一步非常关键，因为它把“时序问题”转成了“节点关系问题”：

- 每个节点代表一个通道在整个暂态过程中的摘要表示
- 节点之间的交互对应通道之间的依赖关系

## 9. 图注意力模块

### 9.1 为什么要把通道当成图节点

电力暂态稳定性不是一个单通道现象。失稳通常来自多个量之间的耦合：

- 不同电压测点的联动
- 不同机组功角的扩散
- 电压和功角之间的联动退化

因此模型在时间编码之后，不直接进入分类头，而是把所有通道节点拼起来：

- `all_nodes = [v_nodes ; d_nodes]`

然后在这些节点上做图注意力。

### 9.2 图注意力的具体做法

每个图层都做这几步：

1. 先做 `GraphNorm`，按节点集合归一化。
2. 用线性层生成 `Q/K/V`。
3. 计算多头注意力分数。
4. 用 `node_mask` 屏蔽不存在的节点。
5. 做 softmax 得到节点间权重。
6. 汇聚消息，经过输出投影和前馈网络，形成新的节点表示。

当前配置里：

- `graph_type: dense_attn`
- `n_graph_layers: 2`
- `n_heads: 4`

这意味着当前使用的是全连接图注意力，不做稀疏 top-k 剪枝。

### 9.3 GraphNorm 的作用

图模块没有直接使用普通 `LayerNorm`，而是用了 `GraphNorm`。它按有效节点集合计算均值和方差，并显式考虑 `node_mask`，这比普通归一化更适合节点数可变、含有 padding 节点的场景。

## 10. 电压分支与功角分支的融合

模型没有把电压和功角粗暴拼接后直接分类，而是设计了一个显式融合模块 `_fuse_modalities`。

融合时会先分别得到：

- 电压节点池化结果 `pooled_v`
- 功角节点池化结果 `pooled_d`
- 所有节点整体池化结果 `pooled_nodes`

然后引入一个可学习的 `risk_token`，和 `pooled_nodes` 相加得到“风险上下文”。之后把：

- `pooled_v`
- `pooled_d`
- `risk_context`

拼起来送入 `gate_mlp`，产生一个 gating 向量：

- `gate = sigmoid(...)`

最终融合方式是：

- `gate * pooled_v + (1 - gate) * pooled_d`

再与 `pooled_nodes` 一起经过投影得到最终融合表示。

这个设计的含义是：

- 模型不会固定认为“电压更重要”或“功角更重要”
- 它可以针对不同样本，自适应决定当前更依赖哪一类信息

比如：

- 有的失稳样本，电压塌陷特征更明显
- 有的失稳样本，功角发散特征更明显

门控融合正是为这种差异设计的。

## 11. 频域分支

当前配置中：

- `use_freq_branch: true`

所以模型除了时域路径外，还额外提取一条频域摘要支路。

具体做法是：

1. 对输入序列按通道做 FFT。
2. 取频谱幅值并做 `log1p`。
3. 在通道维做 mask-aware 平均。
4. 把频谱插值或压缩到固定 `32` 个频率 bin。
5. 将电压频谱和功角频谱拼接后映射到 `d_model`。

最后，频域投影结果会加到融合表示上。

这条分支的意义在于补充时域 patch 编码难以直接表达的信息，比如：

- 振荡强度
- 振荡频率分布
- 恢复过程中的周期性成分

对暂态稳定问题来说，频域信息往往能补充时域峰值和尾部特征。

## 12. 多出口设计

当前模型设置：

- `n_exit = 3`
- `early_exit = true`

实际实现里，模型会在多个层次形成决策状态：

- 初始融合状态
- 每经过一层图注意力后的融合状态

这些状态会被整理成固定数量的出口。如果状态数不足，会通过 `exit_refiner` 继续细化；如果状态数过多，会均匀抽取到 `n_exit` 个。

每个出口都有两个头：

- `cls head` 输出二分类 logits
- `risk head` 输出风险值

这样做的目的有三个：

1. 浅层和深层都能收到监督，训练更稳定。
2. 中间层表示更容易变得可判别。
3. 为未来可能的“提前决策”或在线推理留出结构基础。

虽然当前训练和评估默认使用的是最后一个出口进行最终预测，但所有出口都会参与损失计算。

## 13. 最终输出与辅助表示

模型最终还会输出两个额外结果：

- `emb`
- `channel_importance`

其中：

- `emb` 是最后一个决策状态经过投影并做 L2 归一化后的向量，用于监督对比学习
- `channel_importance` 是节点向量的范数，可视为一个简单的通道重要性摘要

这说明这个模型不只是单纯为了得到 `logits`，还在显式学习一个更结构化的判别表示空间。

## 14. 物理风险特征是如何构造的

训练时，代码不会只看样本的最终 `stable / unstable` 标签，还会从原始时序中构造一组“物理风险特征”。

主要包括：

- `low_v_ratio`
  电压低于 `v_warn=0.9` 的比例
- `danger_v_ratio`
  电压低于 `v_danger=0.85` 的比例
- `tail_low_ratio`
  序列尾部最后 `10%` 时间窗内的低电压比例
- `angle_peak`
  最大功角扩散幅度
- `angle_tail`
  尾部平均功角扩散
- `angle_speed`
  功角变化速度
- `missing_ratio`
  缺失占比

其中功角部分会先自动判断单位是弧度还是角度：

- 如果 `|delta|` 的 95 分位数小于 `8`，就认为更像弧度，乘上 `57.2957795` 转成角度
- 否则视为已经是角度

这一步很实用，因为历史数据常常不保证单位完全统一。

## 15. 风险目标与物理伪标签

代码会把上面的物理特征按加权方式组合成一个连续风险目标：

- `risk_target in [0, 1]`

权重大致体现了这样一个优先级：

- 危险低电压占比最重要
- 一般低电压占比次之
- 尾部低电压、功角峰值、尾部功角扩散、角速度、缺失率也共同参与

然后再基于 `risk_target` 和一些硬阈值构造一个物理伪标签：

- 当 `risk_target >= 0.5`
- 或 `danger_v_ratio >= 0.12`
- 或 `angle_peak >= angle_danger`

则 `phys_label = 1`

这个 `phys_label` 不是替代真实标签，而是给模型一个“从物理统计角度看，这个样本已经很危险”的辅助约束。

## 16. 总损失函数设计

模型的总损失不是单一交叉熵，而是四部分加权求和。

### 16.1 分类损失

分类部分使用的是带类别权重的 `Focal Loss`。

原因是：

- 暂态稳定数据常常类别不平衡
- 普通交叉熵容易被大量简单样本主导
- Focal Loss 会把更多学习重点放到难样本上

当前还支持自动估计 `unstable` 类别权重：

- `class_weight_unstable = max(default_weight, neg / pos)`

这样训练时会更重视少数类。

### 16.2 风险回归损失

每个出口预测的 `risk_hat` 都会和 `risk_target` 做 `Smooth L1 Loss`。这相当于告诉模型：

- 不只是要判断“稳不稳”
- 还要学会估计“有多危险”

这样会让表示空间更连续，减轻硬二分类带来的边界突变问题。

### 16.3 物理一致性损失

这部分约束分类输出、风险输出和物理特征之间保持一致。它包含三项：

1. `prob_unstable` 与 `risk_target` 的 MSE
2. `risk_hat` 与 `risk_target` 的 `Smooth L1`
3. `prob_unstable` 与 `phys_label` 的 BCE

直观上，它在逼迫模型满足这样一种关系：

- 如果物理统计已经很危险，模型的失稳概率和风险输出不能太低
- 如果物理统计风险不高，模型也不应该无依据地给出过高失稳概率

### 16.4 监督对比学习损失

最后一个判别向量 `emb` 会参与 supervised contrastive loss。

核心思路是：

- 同类样本在嵌入空间里更接近
- 异类样本更远离

此外，这里还把 `risk_target` 融入了 pair weighting：

- 同类样本如果风险更相近，就拉得更近
- 异类样本如果风险接近，在 `hard_neg=true` 时会被当作更难的负样本

这一步非常有意思，因为它不是简单按标签拉开，而是让“边界附近的难分样本”在表示空间上接受更细致的约束。

### 16.5 总损失表达

当前总损失可以概括为：

`total = w_cls * cls_loss + w_risk * risk_loss + lambda_phys * phys_loss + lambda_con * con_loss`

当前配置值为：

- `w_cls = 1.0`
- `w_risk = 0.5`
- `lambda_phys = 0.2`
- `lambda_con = 0.1`

可以看出，分类依然是主任务，风险回归次之，物理一致性和对比学习作为结构化辅助约束。

## 17. 训练流程

训练主流程可以概括成下面几步：

1. 读取配置并固定随机种子。
2. 根据 `train_dir` 和 `test_dirs` 构造跨系统协议。
3. 从训练域内部按 `val_ratio` 切出验证集。
4. 只用训练子集拟合归一化器。
5. 构建 train / val / OOD test 的 DataLoader。
6. 根据训练集类别分布自动估计正类权重。
7. 用 `AdamW` 优化模型。
8. 每个 epoch 后在验证集上评估。
9. 用 `f1 + 0.5 * recall + 0.5 * auc` 作为 early stopping 监控分数。
10. 保存 `last.pt` 和当前最佳 `best_by_36val_f1.pt`。
11. 训练结束后重新加载最佳 checkpoint，再统一评估 `36val`、`37test`、`74test`。

这里要注意一个实现细节：

- 训练过程中每个 epoch 保存的是 `36val_latest`
- 训练结束后的正式结果会再输出一次 `36val`
- OOD 的 `37test` 和 `74test` 只在最终最佳模型上统一导出

这保证了最终报告对应的是“最佳验证模型”而不是最后一个 epoch。

## 18. 评估与输出文件

评估阶段默认使用最后一个出口的分类结果，并基于阈值：

- `threshold = 0.5`

生成二分类预测。输出指标包括：

- `accuracy`
- `precision`
- `recall`
- `f1`
- `macro_f1`
- `micro_f1`
- `auc`

此外还会导出：

## 19. 这个模型为什么适合当前任务

从工程和方法两个层面看，这个设计是比较贴合暂态稳定 OOD 任务的。

### 19.1 它不是固定 shape 假设

模型没有要求：

- 固定时间长度
- 固定电压通道数
- 固定功角通道数

这使它适合真实电网数据中常见的系统差异、测点差异和截断长度差异。

### 19.2 它先学单通道时间行为，再学跨通道关系

这是一个比较自然的层次化建模策略：

- 单通道先看自身动态
- 多通道再看交互耦合

比起一上来把所有通道打平，更容易保留物理结构。

### 19.3 它显式分开电压和功角

电压和功角虽然都与稳定性相关，但物理语义并不相同。分模态编码再融合，比一开始生硬拼接更合理，也更有利于跨系统迁移。

### 19.4 它不仅做标签拟合，还做物理一致性约束

如果只做监督分类，模型有可能学到很多“只在训练系统成立”的统计捷径。加入物理风险特征、风险回归和一致性约束，可以逼迫模型向更稳健、更可解释的方向收敛。

### 19.5 它对 OOD 更友好

当前实验不是普通 IID 验证，而是跨系统 OOD。这个模型中的多个设计都在服务这个目标：

- channel-agnostic patch 编码
- mask-aware 聚合
- 图注意力而非固定拓扑卷积
- robust normalization
- 物理辅助监督

它们共同降低了模型对“某个固定系统编号和固定 shape”的依赖。

## 20. 一句话总结

当前这套 `PhysHPGT` 可以概括成：

“一个面向可变长度、可变通道的双模态暂态稳定模型。它先对每个通道做局部时间 patch 编码和时序建模，再把通道当成图节点做跨通道关系学习，同时引入频域摘要、风险回归、物理一致性约束和监督对比学习，以提升跨系统 OOD 泛化能力。”
